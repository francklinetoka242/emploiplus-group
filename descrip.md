# Description du fonctionnement du site EmploiPlus Group

## 1. Objectif du site
EmploiPlus Group est une plateforme de services dédiés à l’emploi, au recrutement et à la mise en relation entre entreprises et talents. Elle permet de :
- publier et diffuser des offres d’emploi ;
- présenter les services de l’entreprise ;
- partager des contenus utiles (blog) ;
- permettre aux visiteurs de découvrir des opportunités professionnelles ;
- offrir un espace d’administration pour gérer le contenu du site.

## 2. Fonctionnement général du site
Le site fonctionne comme une plateforme web publique avec un espace d’administration interne.

### 2.1 Partie publique
Les visiteurs peuvent :
- consulter la page d’accueil ;
- découvrir les services proposés ;
- voir les offres d’emploi publiées ;
- ouvrir le détail d’une offre ;
- consulter les articles du blog ;
- contacter l’équipe via le formulaire de contact ;
- partager une offre sur les réseaux sociaux.

### 2.2 Partie administration
L’administration permet à l’équipe de :
- créer et gérer des offres d’emploi ;
- publier ou masquer des offres ;
- gérer les articles du blog ;
- gérer certains contenus SEO ;
- administrer les membres de l’équipe.

## 3. Parcours utilisateur actuel
### Visiteur simple
1. Un utilisateur arrive sur la page d’accueil.
2. Il peut naviguer vers les offres d’emploi ou les services.
3. S’il trouve une offre intéressante, il peut la consulter.
4. Il peut postuler en suivant le canal prévu (email ou lien externe).

### Administrateur
1. L’administrateur se connecte via une page d’authentification.
2. Il accède à un tableau de bord d’administration.
3. Il publie des offres et gère le contenu du site.

## 4. État actuel du système d’authentification
Le site dispose déjà d’un système de connexion pour l’administration, basé sur un système d’authentification sécurisé.

Cette base peut être étendue pour permettre :
- la création de comptes candidats ;
- la connexion des candidats ;
- la gestion de leurs profils ;
- le suivi de leurs candidatures.

## 5. Fonctionnalités à définir pour la création de compte candidat
La création de compte candidat doit permettre à un utilisateur de créer un profil professionnel et de pouvoir interagir avec la plateforme.

### 5.1 Informations de base du compte
À collecter au moment de l’inscription :
- prénom et nom ;
- adresse email ;
- mot de passe ;
- numéro de téléphone ;
- ville ou région ;
- type de profil (étudiant, salarié, freelance, etc.).

### 5.2 Profil candidat
Après inscription, le candidat devrait pouvoir remplir un profil complet comprenant :
- photo de profil ;
- CV ;
- expérience professionnelle ;
- compétences ;
- formations ;
- langue(s) parlées ;
- disponibilité ;
- salaire souhaité ;
- secteur d’intérêt ;
- type de contrat recherché.

### 5.3 Fonctionnalités de connexion et sécurité
Le système doit inclure :
- inscription ;
- connexion ;
- déconnexion ;
- réinitialisation du mot de passe ;
- vérification de l’email ;
- protection contre les comptes frauduleux ;
- gestion des permissions selon le rôle candidat.

### 5.4 Fonctionnalités liées aux offres
Un candidat connecté devrait pouvoir :
- enregistrer des offres favorites ;
- postuler directement à une offre ;
- suivre l’état de ses candidatures ;
- recevoir des alertes sur de nouvelles offres correspondant à son profil ;
- filtrer les offres selon sa localisation ou son domaine.

### 5.5 Tableau de bord candidat
Le compte candidat peut être organisé autour d’un tableau de bord avec les sections suivantes :
- profil personnel ;
- CV et documents ;
- candidatures envoyées ;
- offres enregistrées ;
- notifications et alertes ;
- paramètres du compte.

## 6. Règles de fonctionnement attendues
La création de compte candidat doit être :
- simple ;
- rapide ;
- intuitive ;
- sécurisée ;
- adaptée au contexte de recrutement ;
- conforme à la protection des données personnelles.

## 7. Propositions de structure fonctionnelle
### Version minimale
- inscription candidat ;
- connexion ;
- profil de base ;
- dépôt de CV ;
- candidature à une offre.

### Version intermédiaire
- profil détaillé ;
- suivi des candidatures ;
- offres favorites ;
- alertes email ;
- mise à jour du profil.

### Version avancée
- recommandation d’offres personnalisées ;
- matching candidat/offre ;
- intégration avec WhatsApp ou email ;
- historique de candidatures ;
- notifications automatiques.

## 8. Conclusion
Le site actuel est déjà orienté vers l’emploi et le recrutement. La prochaine étape consiste à transformer la plateforme en un espace où chaque candidat peut créer un compte, construire son profil, postuler plus facilement et suivre ses démarches de recrutement.

Ce document servira de base pour définir précisément les fonctionnalités à développer pour la création de compte candidat.
