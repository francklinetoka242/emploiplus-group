# src/core

## Rôle

Ce dossier centralise les fondations transverses de l’application : configuration, types partagés, utilitaires communs et primitives d’infrastructure.

## Ce qui doit y être placé

- clients API de base
- configuration d’environnement
- types partagés
- utilitaires génériques
- i18n et SEO partagés

## Ce qui ne doit jamais y être placé

- composants métiers spécifiques à une feature
- logique d’authentification métier
- composants UI visuels
- appels Supabase dédiés à un domaine précis
